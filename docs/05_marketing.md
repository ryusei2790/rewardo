# マーケティング戦略

## 現在のフェーズ

**フェーズ0：個人利用・ポートフォリオ**

現時点では収益化は考えておらず、以下を目的とする：

- 自分が実際に使うプロダクトを作る
- Next.js / Firebase を使った実装経験をポートフォリオとして示す
- 将来的な公開・展開の基盤を作る

---

## 競合・差別化分析

### 主な競合

| サービス | 特徴 | 弱み |
|---------|------|------|
| Todoist | 高機能・多機能 | 続けるモチベーション設計がない |
| Notion | 自由度が高い | 重い・セットアップが面倒 |
| Things 3 | UIが美しい | iOS/Mac限定・高価 |
| Habitica | ゲーミフィケーション | ゲームすぎて疲れる |

### Rewardo の差別化ポイント

| 特徴 | 説明 |
|------|------|
| ご褒美ガチャ | 既存Todoアプリにない「ランダム性によるインセンティブ」 |
| シンプル設計 | 余計な機能を排除。タスク名だけで即使える |
| 自分でカスタマイズ | ご褒美内容・出現確率を自分で設定できる自由度 |
| 心理学的根拠 | 変動比率強化スケジュールに基づいたモチベーション設計 |

## ターゲットペルソナ

### ペルソナ A：社会人（25〜35歳）

- 毎日タスクリストを作るが消化しきれない
- 自己管理・習慣化に関心がある
- シンプルなツールを好む

### ペルソナ B：学生（18〜24歳）

- 課題・勉強のモチベーションが続かない
- ゲーム感覚で取り組める仕組みが欲しい
- スマホ・PCどちらでも使いたい
