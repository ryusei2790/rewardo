# 作成目的

## 背景・課題

「やるべきことはわかっているのに続かない」という問題は多くの人が抱えている。
既存のTodoアプリはタスクを管理する機能はあるが、**続けるためのモチベーション設計**が弱い。

## 解決アプローチ

行動経済学における**変動比率強化スケジュール**の原理を応用する。
スロットマシンやガチャが「いつ当たるかわからないから続けたくなる」のと同じ仕組みで、
タスク完了にご褒美のランダム性を組み合わせることで、自然に行動を促す。

> 「次のタスクを終えたらご褒美が出るかも」という期待感がモチベーションになる。

## 目的

- 個人のタスク消化率・習慣化を改善する
- シンプルかつ続けやすいUXを提供する
- ポートフォリオとして技術スタックへの習熟を示す

## 個人的な動機

- Next.js / Firebase / dnd-kit など実践的なスタックを組み合わせて開発経験を積む
- 自分自身が実際に使いたいプロダクトを作る
