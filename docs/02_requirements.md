# アプリ要件定義

## ターゲットユーザー

- 個人の生産性・習慣化に課題を感じる社会人
- 勉強・課題をこなすモチベーションが欲しい学生

## 利用モード

| モード | 説明 | データ保存先 |
|--------|------|------------|
| ゲストモード | ログインなしで即利用可能 | ローカルストレージ |
| ログインモード | Firebaseアカウント（Google）でログイン | Firestore（クラウド） |

- トップページでどちらか選択可能
- ゲスト → ログインへ切り替えた場合、ローカルデータの引き継ぎは**なし**（新規スタート）

## ページ構成

| ページ | パス | 説明 |
|--------|------|------|
| トップ | `/` | モード選択（ゲスト / ログイン） |
| ホーム | `/home` | タスク管理・ご褒美ポップアップ |
| ご褒美設定 | `/reward-settings` | ご褒美・パターン登録 |
| ログイン | `/login` | Google認証 |

## 機能要件

### ホーム画面

| 機能 | 詳細 |
|------|------|
| タスク追加 | タスク名を入力して追加（タスク名のみ・シンプル構成） |
| タスク一覧 | 登録済みタスクを表示順に一覧表示 |
| ドラッグ並び替え | dnd-kitでタスクの順序を変更。ゲスト：ローカルストレージ更新、ログイン：DBのposition更新 |
| チェックで完了 | チェックを入れると完了済みに。完了済みは下部に薄表示 |
| ご褒美ポップアップ | 条件達成時にconfettiアニメーション + ご褒美内容を表示 |

### ご褒美設定ページ

| 機能 | 詳細 |
|------|------|
| ご褒美登録 | 内容（テキスト）と重み（1〜10）を設定して登録 |
| ご褒美一覧・削除 | 登録済みご褒美の確認・削除 |
| パターン登録 | ご褒美が出るタスク完了数の幅を指定（min〜max） |
| パターン一覧・削除 | 複数パターンの確認・削除 |

### 認証

| 機能 | 詳細 |
|------|------|
| ログイン | Googleアカウントでワンクリックログイン（Firebase Auth） |
| ログアウト | セッション終了 |
| 認証ガード | 未ログイン時は `/login` へリダイレクト |

## ご褒美ロジック詳細

```
1. タスクを完了するたびに done_count を +1
2. done_count が target_count に達したらご褒美発動
3. target_count の決め方：
   - 登録された reward_patterns からランダムに1つ選ぶ
   - 選んだパターンの min〜max の整数をランダムで target_count に設定
   - 例）パターン「3〜5」→ 3, 4, 5 のどれか
4. ご褒美の選び方：
   - 登録された rewards を weight（重み）に応じた加重抽選
   - 例）重み3のご褒美は重み1の3倍出やすい
5. ご褒美ポップアップ表示（confetti演出）
6. done_count をリセット → 次の target_count を再抽選
```

## 非機能要件

| 項目 | 内容 |
|------|------|
| 利用形態 | 個人利用のみ（チーム・共有機能なし） |
| タスク属性 | タスク名のみ（期限・カテゴリなし） |
| 運用単位 | 1日単位を想定 |
| デザイン | 落ち着いた大人向け（ダークトーン・モノクロベース） |
| セキュリティ | ログインモード：Firestoreセキュリティルールにより他ユーザーのデータにアクセス不可 |
| 認証 | 任意。ゲストモードはローカルストレージのみで動作 |
| データ永続化 | ゲスト：ブラウザ依存（クリアで消失）/ ログイン：クラウド永続 |
